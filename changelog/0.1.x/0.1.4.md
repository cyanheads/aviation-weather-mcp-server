# 0.1.4 — 2026-06-12

## Added

- **`createApp()` identity** — explicit `name` and `title`, both set to `aviation-weather-mcp-server`, so the hyphenated repo name is the display identity on every surface instead of relying on the npm-scoped package name.
- **`scripts/clean-mcpb.ts`** — post-pack bundle cleaner wired into the `bundle` script: runs `mcpb clean`, then strips dependency-shipped agent docs (`node_modules/**/skills/`, `.claude/`, `.agents/`, stray `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach, and asserts zero remain.
- **`Dockerfile` healthcheck** — `HEALTHCHECK` probes `/healthz` via a bun-native `fetch` (the slim image ships no curl/wget). `ARG APP_VERSION` now stamps `org.opencontainers.image.version`.
- **`lint-packaging` bundle-content checks** — `.mcpbignore` must use root-anchored patterns and not strip critical runtime paths; a built `.mcpb` under `dist/` must contain zero `node_modules` agent-doc entries. `AGENT_DOC_ENTRY` is shared with `clean-mcpb.ts` (a test asserts the two stay identical).
- **`check-framework-antipatterns`** — new `coerce-boolean-env-flag` rule flags `z.coerce.boolean()` on env flags (`Boolean("false") === true`; use `z.stringbool()`); comment lines are skipped so the rule stays sound when docs name the pattern.

## Changed

- **`aviation_find_stations` / `aviation_get_pireps`** — `missing_search_criteria` and `missing_location` now use `JsonRpcErrorCode.ValidationError` instead of `InvalidParams`.
- **`.mcpbignore`** — dev-directory and dev-file patterns root-anchored with a leading `/` (`/.claude/`, `/skills/`, `/Dockerfile`, …) so they match only at bundle root.
- **`bundle` script** — builds via `bun run build` and runs `clean-mcpb.ts` on the packed `.mcpb`.
- **Vendored skills** — `skills/` resynced to the `@cyanheads/mcp-ts-core` `0.10.6` set; adds the `techniques` reference skill (outline-on-overflow).
